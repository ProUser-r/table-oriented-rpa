# Customizing Workflows for Your Email Provider

This guide shows how to customize workflows for your specific email setup.

## Gmail Setup

### Prerequisites
- Gmail account
- Allow "Less secure app access" or create an App Password
- Google Chrome installed

### Step 1: Login Handling

If you need to login automatically, add before email steps:

```yaml
- id: "login_email"
  name: "Enter email"
  action: "type"
  params:
    text: "${env.GMAIL_USER}"

- id: "click_next"
  name: "Click Next"
  action: "click"
  params:
    locator: "text:Next"

- id: "wait_password"
  name: "Wait for password field"
  action: "wait"
  params:
    seconds: 2

- id: "login_password"
  name: "Enter password"
  action: "type"
  params:
    text: "${env.GMAIL_PASS}"

- id: "click_signin"
  name: "Click Sign in"
  action: "click"
  params:
    locator: "text:Sign in"
```

### Step 2: Find Your Email

Update the sender/subject to match your actual emails:

```yaml
- id: "find_email"
  name: "Find email"
  action: "click_email"
  params:
    # Option 1: Search by sender
    from_sender: "your-partner@company.com"
    # OR Option 2: Search by subject
    subject: "Monthly Report"
    # OR Option 3: Create image reference
    locator: "image:references/gmail_your_email.png"
```

### Step 3: Configure Download Directory

```yaml
- id: "download"
  name: "Download attachments"
  action: "download_attachments"
  params:
    download_dir: "./gmail_tables"  # Adjust path as needed
    pattern: "*.png"                # Match your image format
```

### Step 4: Set Environment Variables

Create `.env` file:
```bash
GMAIL_USER=your-email@gmail.com
GMAIL_PASS=your-app-password
RPA_GUI_WORKFLOW_DIR=./workflows
```

## Outlook Setup

### Prerequisites
- Microsoft 365 account with Outlook Web Access
- Modern browser (Chrome, Edge)
- Access to outlook.office.com

### Step 1: Open Outlook Web Access

```yaml
- id: "open_outlook"
  name: "Open Outlook"
  action: "open_browser"
  params:
    url: "https://outlook.office.com"
    headless: false
```

### Step 2: Wait for Login

Outlook may require login:

```yaml
- id: "wait_login"
  name: "Wait for login (manual or auto)"
  action: "wait"
  params:
    seconds: 5  # Adjust if needed
```

### Step 3: Find Email

Different for Outlook - try by subject:

```yaml
- id: "find_outlook_email"
  name: "Open email"
  action: "click_email"
  params:
    subject: "Your expected subject"
```

Or use text search in Outlook UI first, then image locator.

### Step 4: Download from Outlook

```yaml
- id: "outlook_download"
  name: "Download Outlook attachments"
  action: "download_attachments"
  params:
    download_dir: "./outlook_attachments"
    pattern: "*.png"
    timeout: 30
```

## Creating Reference Images

### Why Use Reference Images?

More reliable than text matching in unstable UI.

### How to Create

1. **Take screenshot manually:**
   - Run workflow step by step manually
   - When email is open, take screenshot
   - Use screenshot tool to capture element

2. **Save to references directory:**
   ```
   references/
   ├── gmail/
   │   ├── inbox.png
   │   ├── email_entry.png
   │   └── send_button.png
   └── outlook/
       ├── inbox.png
       └── download_icon.png
   ```

3. **Use in workflow:**
   ```yaml
   - id: "click_email"
     name: "Click email"
     action: "click"
     params:
       locator: "image:references/gmail/email_entry.png"
   ```

### Tips

- **Keep it small**: Crop to just the element, not entire screen
- **High contrast**: Avoid similar-colored backgrounds
- **Consistent UI**: Test on same resolution/browser
- **Update regularly**: UI changes may require new images

## Word Document Setup

### Create Template

1. **Open Word**
2. **Add placeholder text:**
   ```
   Analysis Report
   ================
   
   <!-- REPORT_HERE -->
   
   Generated by RPA System
   ```

3. **Save as .docx**

### Update Workflow

```yaml
- id: "open_template"
  name: "Open template"
  action: "open_word"
  params:
    path: "C:\\Your\\Path\\template.docx"

- id: "insert"
  name: "Insert report"
  action: "insert_text"
  params:
    search_for: "<!-- REPORT_HERE -->"
    text: "${analysis.report}"

- id: "save"
  name: "Save"
  action: "save_document"
  params:
    wait_before_save: 1
```

### Output Path

Adjust in `.env`:
```bash
REPORT_OUTPUT_PATH=C:\Users\YourName\Documents\Reports\report.docx
```

## Complete Custom Workflow Template

```yaml
name: "My Custom Email Processing"
description: "Process my specific emails"

config:
  timeout: 120
  browser_headless: false

steps:
  # Email Phase
  - id: "open_email"
    name: "Open my email provider"
    action: "open_browser"
    params:
      url: "https://mail.google.com"
      headless: false

  - id: "wait_load"
    name: "Wait for email to load"
    action: "wait"
    params:
      seconds: 5

  - id: "find_my_email"
    name: "Find specific email"
    action: "click_email"
    params:
      from_sender: "partner@mycompany.com"

  - id: "get_attachments"
    name: "Download images"
    action: "download_attachments"
    params:
      download_dir: "./my_images"
      pattern: "*.png"

  - id: "close_email"
    name: "Close email"
    action: "close_application"
    params:
      app: "current"

  # Analysis Phase
  - id: "analyze"
    name: "Analyze tables"
    action: "process_with_robot"
    params:
      image_dir: "./my_images"
      output_dir: "./results"
      store_result: "results"

  # Report Phase
  - id: "open_report"
    name: "Open Word template"
    action: "open_word"
    params:
      path: "C:\\Users\\YourName\\Documents\\template.docx"

  - id: "write_report"
    name: "Insert analysis"
    action: "insert_text"
    params:
      search_for: "<!-- ANALYSIS -->"
      text: "${results.report}"

  - id: "save_report"
    name: "Save"
    action: "save_document"
    params:
      wait_before_save: 1

  - id: "close_word"
    name: "Close Word"
    action: "close_application"
    params:
      app: "current"
```

## Testing Your Workflow

### 1. Dry Run

Test with the example workflows first:
```bash
python examples/gui_workflow_example.py
```

### 2. Manual Steps

Before running full workflow, test each component:
1. Can you access your email provider?
2. Can you download attachments manually?
3. Can you process images with TableRobot?
4. Does Word template exist?

Check `logs/workflow.log` for any issues

### 4. Screenshots

Review screenshots in `logs/[workflow_name]/` to verify each step

## Common Issues & Fixes

### Can't Find Email

**Problem:** Email not found by sender or subject

**Solutions:**
1. Use exact email address/subject
2. Create image reference of the email
3. Add manual search step
4. Check email is in current view

### Attachments Not Downloading

**Problem:** Files not found in download directory

**Solutions:**
1. Verify email has attachments
2. Check file pattern (*.png vs *.jpg)
3. Increase timeout
4. Check downloads folder permissions

### Word Template Not Found

**Problem:** open_word step fails

**Solutions:**
1. Verify full path is correct
2. Use forward slashes: `C:/path/to/file.docx`
3. Test path in explorer first
4. Ensure .docx format, not .doc
